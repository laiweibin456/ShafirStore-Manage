# 图片上传功能使用指南

## 一、功能概述

商品管理系统已集成图片本地上传功能，基于 Nginx 作为图片存储服务器，实现商品图片的上传、存储和访问。

## 二、文件结构

新增/修改的文件：

### 后端 (Java Spring Boot)
- `controller/UploadController.java` - 图片上传控制器
- `service/UploadService.java` - 上传服务接口
- `service/impl/UploadServiceImpl.java` - 上传服务实现
- `config/SecurityConfig.java` - 添加 `/api/upload/**` 路径放行
- `resources/application.yml` - 添加上传配置

### 前端 (Vue 3 + Element Plus)
- `components/ImageUpload.vue` - 图片上传组件
- `api/upload.js` - 上传 API 接口
- `views/product/ProductManage.vue` - 商品管理页面集成上传组件
- `vite.config.js` - 添加图片代理配置

### 配置文件
- `nginx-image.conf` - Nginx 图片服务器配置

## 三、部署步骤

### 1. 创建图片存储目录

在 Linux 服务器上创建图片存储目录：

```bash
sudo mkdir -p /www/shafirstore/images/products
sudo chmod 755 /www/shafirstore/images/products
```

### 2. 配置 Nginx

将 `nginx-image.conf` 文件复制到 Nginx 配置目录：

```bash
sudo cp nginx-image.conf /etc/nginx/conf.d/shafirstore-image.conf
```

修改配置文件中的路径和端口（如果需要）：

```nginx
# 图片存储根目录 - 确保与 application.yml 中的 upload.path 一致
alias /www/shafirstore/images/products/;

# 图片访问基础 URL - 确保与 application.yml 中的 upload.base-url 一致
# http://your-server-ip:8081/images/products/xxx.jpg
```

测试并重载 Nginx 配置：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

### 3. 修改 application.yml 配置

根据实际部署环境修改以下配置：

```yaml
upload:
  path: /www/shafirstore/images/products  # Linux 服务器上的存储路径
  base-url: http://localhost:8081/images/products  # Nginx 访问地址
```

### 4. 配置跨域访问（如需要）

如果前端和后端部署在不同域名下，需要确保：
- Nginx 配置中已添加 CORS 头
- Spring Boot 的 CorsConfig 已正确配置

## 四、使用功能

### 商品图片上传

1. 登录商品管理系统
2. 进入商品管理页面
3. 点击"新增商品"或"编辑"已有商品
4. 在商品表单中，点击图片上传区域
5. 选择本地 JPG、PNG、GIF 或 WebP 格式的图片（最大 10MB）
6. 上传成功后，图片 URL 会自动保存到商品信息中

### 图片访问

上传成功的图片可以通过以下地址访问：

```
http://localhost:8081/images/products/{filename}
```

在开发环境中，Vite 代理会处理图片请求。

## 五、安全措施

### 前端安全验证
- 文件类型验证：仅允许 JPG、PNG、GIF、WebP 格式
- 文件大小限制：最大 10MB
- 格式验证在前端和后端双重检查

### 后端安全验证
- Multipart 文件大小限制（application.yml 中配置）
- 文件类型白名单验证
- 文件扩展名校验
- 文件名随机化（UUID），防止文件覆盖和路径遍历攻击

### Nginx 安全配置
- 限制客户端最大请求体大小为 10MB
- 图片缓存配置（7天）
- 禁止访问上传目录之外的文件

## 六、常见问题

### 1. 上传失败，提示 "上传目录不存在"
确保 `/www/shafirstore/images/products` 目录已创建且有写入权限。

### 2. 图片上传成功但无法访问
- 检查 Nginx 服务是否启动
- 检查防火墙是否开放 8081 端口
- 确认 `application.yml` 中的 `base-url` 与 Nginx 配置一致

### 3. 上传大图片失败
- 检查 `application.yml` 中 `spring.servlet.multipart.max-file-size` 配置
- 检查 Nginx 配置中 `client_max_body_size` 设置

### 4. 跨域访问图片失败
确保 Nginx 配置中添加了 CORS 响应头：
```nginx
add_header 'Access-Control-Allow-Origin' '*' always;
```

## 七、开发环境测试

1. 启动后端服务：`mvn spring-boot:run` 或通过 IDE 启动
2. 启动前端服务：`npm run dev`
3. 启动 Nginx 图片服务（开发环境可跳过，前端已配置代理）

注意：开发环境中图片通过 Vite 代理访问，无需单独启动 Nginx。但如果要测试完整的图片访问流程，需要启动 Nginx。
