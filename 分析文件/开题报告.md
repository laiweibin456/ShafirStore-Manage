毕业设计（论文）开题报告

基于 Spring Boot 的莎菲尔菓子门店管理系统设计与实现

一、研究的背景和意义
（一）研究背景
在烘焙行业数字化转型加速背景下，中小型烘焙门店面临发展机遇与管理滞后的矛盾。多数门店依赖手工记录、Excel 表格运营，在商品管理、库存控制、销售统计、会员维护等核心环节痛点突出：商品上新繁琐、库存更新不及时导致积压或短缺、销售数据统计滞后、会员信息分散难以精准营销。莎菲尔菓子作为典型中小型烘焙门店，传统管理模式已无法适配数字化时代对运营效率、服务质量及决策科学性的要求。
在此背景下，Spring Boot 框架凭借 “约定优于配置” 特性，成为轻量化企业级应用开发优选，具备开发效率高、配置简单、可扩展性强的优势；Vue.js 可高效实现多终端响应式交互界面；MySQL 数据库以强大的事务支持、数据安全性及易用性，满足门店核心数据存储需求。基于 “Spring Boot+Vue.js+MySQL” 技术组合，可为莎菲尔菓子及同类门店提供轻量化、高适配性的管理系统解决方案，破解传统模式的效率瓶颈与数据协同问题。
（二）研究意义
本研究的意义兼具理论价值与实践价值，二者相互支撑、相辅相成。
在理论层面，本研究聚焦 Spring Boot 框架在中小型零售门店管理场景的工程化应用验证，深入探索 Spring Boot 与 MyBatis-Plus、Spring Security、JWT 等技术的整合方案，明确 Vue.js 响应式界面与后端 RESTful API 的交互逻辑，系统梳理 “前端 Vue.js 开发 + 后端 Spring Boot 服务 + 关系型数据库” 的前后端分离架构设计规范与最佳实践。通过本研究，可丰富 Spring Boot 技术在中小型零售行业应用的案例库，为同类门店管理系统的技术选型提供理论参考与方法论支撑，深化对轻量化架构适配中小型门店业务需求的内在机理认知。
在实践层面，本研究通过设计并实现 “莎菲尔菓子门店管理系统”，精准定位中小型烘焙门店的核心需求，提供 “商品 - 库存 - 销售 - 会员 - 权限” 的全流程管理解决方案。系统采用前后端分离开发模式，后端基于 Spring Boot 构建标准化 RESTful API 服务，前端通过 Vue.js 实现响应式交互界面，各模块数据实时联动，不仅能简化门店日常运营流程、减少手工操作误差，提升管理效率与决策科学性，还能优化会员消费体验、增强客户粘性。此外，本研究形成的需求分析方法、数据库设计方案、API 接口规范及部署指南，可直接为同类中小型烘焙门店的数字化转型提供参考，降低技术门槛与实施成本，具备较强的工程应用价值。
（三）国内外研究现状
当前门店管理系统呈现 “大型连锁解决方案与中小型轻量化系统并存” 的格局，主要分为三类：一是大型连锁专用系统（如美团餐饮管理系统、Toast POS），功能全面但架构厚重、部署成本高、冗余度高，二次开发门槛高，不适配中小型门店需求；二是通用型小微企业工具（如有赞零售、Square），轻量化但缺乏烘焙行业个性化适配，模块间数据联动弱；三是 Spring Boot 定制化系统，架构轻量化、开发效率高，但对烘焙门店短保质期商品管理、原料与成品关联等个性化场景挖掘不足，实用性有待提升。
国内系统多聚焦大型连锁，小微企业工具功能单一；国外系统注重模块化与用户体验，但受地域差异影响适配性不足、本土化成本高。本研究的差异化优势的是：场景适配性强，聚焦烘焙行业个性化需求；技术栈轻量化，兼顾开发效率与部署便捷性；功能聚焦核心，操作简单易懂，适配中小型门店人员水平，可填补现有产品空白。
二、研究的基本内容及拟解决的主要问题
（一）研究的基本内容
本选题旨在通过挖掘并分析中小型烘焙门店的运营模式与管理需求，研究门店商品管理、库存控制、销售统计、会员维护等业务流程和管理逻辑，通过轻量化的前后端分离技术手段，优化核心业务流程，构建一个轻量、高效、易操作的莎菲尔菓子门店管理系统，以解决当前传统管理模式效率低下、数据分散、决策滞后等问题，提升门店运营效率与服务质量。
1．设计思路
(1)需求分析：实地调研莎菲尔菓子门店运营流程，深度访谈店长与一线员工，结合同类烘焙门店管理痛点，明确系统功能与非功能需求。功能需求涵盖商品、库存、销售、会员、员工权限、统计分析等核心模块；非功能需求聚焦响应速度、跨设备兼容性、数据安全性、操作便捷性及系统可维护性。
(2)技术选型：依据需求确定最优技术组合。前端采用 Vue.js+HTML5+CSS3+JavaScript，构建响应式交互界面，适配 PC 端、平板等多终端；后端以 Java+Spring Boot 为核心，整合 MyBatis-Plus 实现数据持久化，通过 Spring Security+JWT 保障身份认证与授权；数据库选用 MySQL，满足核心业务数据存储与事务处理需求；辅助工具采用 Maven 保障依赖版本一致，通过 Nginx 部署前端静态资源。
(3)系统设计：基于功能需求与技术选型，采用 B/S 模式三层架构（前端表现层、后端业务逻辑层、数据存储层）进行整体设计；完成系统总体架构图、模块交互图、E-R 图及数据库逻辑结构设计，确保系统具备良好的可扩展性、稳定性与安全性。
(4)模块划分：将系统拆解为后端服务、前端交互、数据持久化等功能模块。后端服务模块负责核心业务逻辑处理与 API 开发，前端交互模块实现用户界面展示与交互，数据持久化模块承担数据存储与操作，各模块通过规范接口协同工作。
(5)接口设计：提供 RESTful API 接口，覆盖商品管理、库存操作、销售统计、会员管理等核心场景；添加基于角色的接口权限控制机制，仅允许授权人员执行敏感操作，保障接口安全性。
(6)测试与部署：开发过程中采用黑盒测试法，开展功能、兼容性及性能测试，验证模块功能合规性，确保系统在不同设备、浏览器及高并发场景下稳定运行；根据测试结果优化系统，编写详细部署指南，明确前后端服务部署步骤，支持本地与云端部署。
2．系统功能模块
系统需实现包括商品管理、库存管理、销售管理、会员管理、员工权限管理、统计分析等功能，具体分析如下：
(1)前端功能
角色登录与权限控制：支持店长、店员等不同角色登录，根据角色分配相应权限，店长可执行全功能操作，店员仅可进行商品销售、库存查询等基础操作，确保系统安全性与数据保密性。
商品管理：支持商品分类管理、商品信息（名称、价格、保质期、图片等）的增删改查，实现商品快速上新与信息维护。
库存管理：展示原料与成品库存明细，支持库存入库、出库记录，设置低库存预警阈值，实现库存短缺自动提醒。
销售管理：支持快速下单、订单查询与统计，自动关联商品库存数据，实现销售数据实时更新。
会员管理：支持会员信息查询、积分管理、消费记录查询，实现会员精准识别与营销。
统计分析：以可视化图表展示销售数据、库存数据、会员消费数据，为经营决策提供数据支持。
登录状态保持：实现登录状态的本地存储，避免重复登录，提升用户使用便捷性。
(2)后端功能
数据存储与管理：存储商品、库存、销售、会员、员工等核心数据，通过 MySQL 实现数据的高效存储与关联管理，确保数据完整性和准确性。
业务逻辑处理：处理用户登录验证、权限分配、商品与库存联动、销售数据统计、会员积分核算等核心业务逻辑，实现系统业务流程的顺畅流转。
API 接口服务：提供 RESTful API 接口，支撑前端与后端的各类数据交互需求，包含商品管理、库存操作、销售统计、会员管理等接口，保障接口的稳定性与安全性。
跨域通信处理：解决前后端分离架构下的跨域问题，确保前端静态资源服务与后端 API 服务之间的顺畅数据交互。
操作日志记录：记录关键操作（如价格修改、商品删除、库存调整）的日志信息，便于追溯与审计。
（二）拟解决的主要问题
结合研究内容与技术选型，本研究需重点解决以下四大核心问题：
1．技术栈整合与架构适配问题：核心解决 Spring Boot 与 MyBatis-Plus、Spring Security、JWT 的高效集成，明确后端模块化配置方案（数据库连接、路由注册、拦截器封装），确保架构清晰可扩展；制定标准化 API 接口规范（含请求方法、参数格式、响应格式、错误处理），解决前后端跨域通信问题，实现顺畅数据交互。
2．业务数据联动与一致性问题：针对烘焙门店商品、库存、销售的强关联关系，设计合理的数据库关联结构与事务处理机制，确保销售下单后库存自动扣减、会员消费后积分实时更新、库存预警及时触发，避免数据不一致引发的经营风险；优化数据查询逻辑，提升多模块联动查询响应速度。
3．前后端分离架构下的权限控制问题：实现多角色精准权限管控，后端基于 Spring Security+JWT 构建令牌认证机制，精准识别用户角色并控制接口访问权限；前端按角色动态展示界面元素，确保功能可见性与权限匹配；建立敏感操作日志记录机制，保障系统操作可追溯。
4．轻量化适配与用户体验优化问题：适配中小型门店低成本部署与简易操作需求，优化部署流程，支持本地与云端轻量化部署，降低硬件配置要求；前端界面遵循 “简洁易用” 原则，简化操作流程、减少冗余步骤，适配门店员工操作习惯；通过数据缓存、查询语句优化等方式，提升系统响应速度，保障页面加载与数据处理高效流畅。
三、研究的方法和技术路线
（一）研究的方法
本研究严格遵循软件工程方法论，结合多种研究方法开展，确保研究过程科学、研究成果可靠，具体方法如下：
1．文献研究法：系统梳理门店管理系统开发、Spring Boot 框架应用相关文献、技术文档及开源项目，重点研究 Spring Boot 与 Vue.js 整合方案、权限控制机制、数据联动处理及中小型零售系统设计思路，为技术选型与架构设计提供理论支撑。
2．调研分析法：实地走访莎菲尔菓子门店，与店长、一线员工深度访谈，收集一手业务流程与需求数据；调研美团餐饮管理系统、有赞零售等主流烘焙门店管理系统，总结其优劣，为需求分析与功能定义提供依据。
3．原型开发模型：采用迭代式开发模式，将系统开发划分为多个 “需求细化 - 设计 - 开发 - 测试” 闭环周期。先开发用户登录、商品管理、基础销售等核心功能原型，验证技术栈可行性；再逐步迭代完善库存管理、会员管理等模块，及时响应需求调整，提升开发效率。
4．实验法：核心模块开发中，通过对比实验验证技术方案合理性。例如对比 Session 与 JWT 两种权限认证方案，选择适配前后端分离架构的最优解；对比不同数据查询优化策略，提升多模块联动查询性能。
5．黑盒测试法：系统开发完成后，不关注内部实现逻辑，仅依据需求规格说明书，设计测试用例验证系统功能完整性、交互流畅性、兼容性及性能指标，确保系统符合预期目标。
（二）研究的技术路线
技术选型：确定全栈技术组合，前端采用 HTML5、CSS3、JavaScript、Vue.js 构建多终端适配的响应式交互界面；后端以 Java+Spring Boot 为核心，整合 MyBatis-Plus 实现数据持久化，通过 Spring Security+JWT 保障身份认证与授权；选用 MySQL 作为关系型数据库存储核心业务数据，搭配 Maven 进行包管理、Nginx 部署前端静态资源，借助 Postman 开展接口测试。
需求与设计：梳理门店核心业务需求，明确功能模块与非功能要求；采用 B/S 模式三层架构完成系统整体设计，绘制模块交互图、E-R 图，设计数据库逻辑结构；制定 RESTful API 接口规范，明确请求方法、参数格式、响应格式及错误处理规则。
后端开发：搭建 Spring Boot 后端项目架构，完成数据库连接、路由注册、拦截器封装等配置；定义商品、库存、销售、会员、员工五大核心数据模型；开发各模块业务逻辑与 API 接口，实现数据存储、权限分配、数据联动等核心功能；解决跨域通信问题，添加操作日志记录机制。
前端开发与集成：开发前端页面静态结构与响应式样式，实现角色登录、商品管理、库存操作、销售下单等交互功能；对接后端 RESTful API 接口，完成前后端数据异步交互；根据用户角色动态展示界面元素，实现登录状态本地存储优化。
测试优化：采用黑盒测试法开展功能测试、兼容性测试与性能测试；针对测试发现的问题，优化数据库查询逻辑、API 响应速度、界面适配效果及操作流程；完善数据缓存机制，提升系统多终端适配性与运行稳定性。
部署总结：制定本地与云端轻量化部署流程，明确运行环境配置步骤；完成系统部署验证，整理项目技术文档与开发资料；梳理研究过程与成果，准备相关答辩材料。
