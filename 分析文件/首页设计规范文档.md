> 版本：v2.0  
> 更新日期：2026-04-10  
> 参考设计：霸王茶姬小程序  
> 设计风格：清新自然、现代简约

---

## 一、设计概述

### 1.1 设计理念

本次首页 redesign 参考了霸王茶姬小程序的设计风格，以**清新自然**为核心设计理念，打造符合现代审美的烘焙品牌小程序。

**核心关键词：**
- 🌿 自然清新
- 🎨 现代简约
- ✨ 高级质感
- 📱 用户友好

### 1.2 设计目标

1. **提升视觉吸引力** - 采用自然绿色调，打造品牌识别度
2. **优化信息层次** - 清晰的功能分区，提升用户体验
3. **增强交互体验** - 流畅的视觉引导，直观的操作反馈
4. **保证响应式效果** - 适配不同尺寸屏幕

## 五、组件设计

### 5.1 用户卡片

**结构：**
```
┌──────────────────────────────────────────────┐
│ [头像]  你好，张三  [会员等级]                │
│         享受专属优惠                         │
│                                              │
│                    180      ¥3500.00         │
│                   积分      累计消费         │
└──────────────────────────────────────────────┘
```

**样式要点：**
- 白色背景，圆角 24rpx
- 阴影：`0 8rpx 32rpx rgba(93, 155, 108, 0.15)`
- 左侧：头像 + 欢迎语 + 会员标签
- 右侧：积分 + 累计消费数据

### 5.2 功能卡片

**结构：**
```
┌─────────────────────────────┐
│ 🏪  门店自取          ›     │
│     提前预定 到店即取        │
└─────────────────────────────┘
```

**样式要点：**
- 左侧色条标识功能类型
  - 自取：绿色 `#5D9B6C`
  - 外卖：橙色 `#F5A623`
- 大图标 + 标题 + 描述
- 右侧箭头引导点击

```
┌────────┐
```

- 96rpx × 96rpx 圆角方框


## 六、交互说明

### 6.1 点击反馈

```css
/* 所有可点击元素添加按压效果 */
.activity:active {
  opacity: 0.7;
  transform: scale(0.98);
}

/* 卡片点击反馈 */
.card:active {
  background-color: #F5F5F5;
}
```

### 6.2 加载状态

- 使用骨架屏 (Skeleton Screen)
- 渐进式加载
- 加载时显示灰色占位块

### 6.3 空状态

- 显示友好的空状态图标
- 提供引导操作按钮
- 使用温和的提示文案

### 6.4 错误处理

- Toast 提示错误信息
- 显示重试按钮
- 保持页面基本结构

## 七、响应式设计

### 7.1 适配策略

```css
/* 使用 rpx 单位自动适配 */
/* 1rpx = 0.5px (iPhone 6) */

/* 安全区域适配 */
padding-top: env(safe-area-inset-top);
padding-bottom: env(safe-area-inset-bottom);

/* 最小字号限制 */
/* iOS: 最小 11pt (约 22rpx) */
/* Android: 最小 10sp */
```

### 7.2 布局适配

| 屏幕宽度 | 每行项目数 | 卡片宽度 |
|---------|-----------|---------|
| < 320px | 2 项 | 46% |
| 320-375px | 4 项 | 23% |
| > 375px | 4 项 | 23% |

```css
/* 响应式图片 */
image {
  width: 100%;
  height: auto;
  object-fit: cover;
```

## 八、性能优化

### 8.1 图片优化

- 使用 WebP 格式
- 压缩图片质量 (80%)
- 懒加载非首屏图片

### 8.2 样式优化

- 避免深层嵌套选择器
- 使用 CSS 变量
- 减少重绘重排

### 8.3 渲染优化

- 首屏内容优先渲染
- 使用虚拟列表
- 避免频繁 setData
